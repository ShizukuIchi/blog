React Native with Expo

發表於 2018-04-09 | 分類於 Javascript

對於很多開發者來說，對於未知領域的環境設定，是個大難點，Expo 將一切打包好，讓開發者能夠近乎零設定就能開始 Coding!

※ 手機版本必須是 Android 4.4 或是 iOS 9.0 以上

為何使用 Expo

  • 不需接觸 native code 如 Java，若遇到一定要修改 native code 的情況則要 detach，Expo 一直致力於功能擴充，好用、常見的套件通常會被涵蓋
  • 呈上點的好處是可直接在網頁、文字編輯器開發 app，也意味不需 Xcode 就能開發 iOS app!
  • 不需 Xcode、Android Studio 就能打開 app，也能一鍵製作 apk、ipa

Get Started

先準備好一些東西

  1. 最新的 Node.js
  2. Expo 圖形介面工具 XDE (本文使用 2.22.1版)
  3. 手機端 app (Expo Client) 或是模擬器
  4. React Native Debugger (optional)

動手囉

打開 Expo XDE

  • 按下 Create new project …

    • 選擇 Tab Navigation 有助於了解簡單 app 之 code 架構

    • 選擇 Blank 開啟空白專案

      選擇資料夾位置以及專案名稱後，開始下載必要檔案，之後 XDE 會自動開啟專案，記得要在該資料夾內執行npm install。

  • 程式碼 Host 方式

    • 預設是 Tunnel，按下右上角的 Share 後，透過 QR Code 就可以在實體手機開啟 app (Expo Client)。
    • 建議改為 LAN，讓手機和電腦處於同一個 Wi-Fi 底下，速度較快。
※ 若使用實體手機 Wi-Fi 訊號需夠強，不然會卡卡的 debug 很痛苦...

開始 Coding

  • 試試開啟 app
手機打開 Expo Client 掃描 QR Code，就會開始打包專案的 code，完成後就會開啟 app，第一個畫面會提示可以修改檔案讓 app 產生變化。

  • 開始編輯 app

    改變 code 時 reload 的方式有兩種：

    • Live Reload，整個 app 自動重新整理。
    • Hot Reload，保持原狀態，將更改的部分新增進 app。

    • Reload JS Bundle，顧名思義重新打包、持行 code。

      一般來說時常微調畫面，Hot Reload 是最合適、最省時的。

Debug

  • app 的 log 會顯示在 XDE 右側的框框內。
  • 使用 React Native Debugger 擁有更棒的 Debug 體驗 (RN Debugger 設定)。

重要套件

react-redux

react 中，更改畫面是由各個 component 的 state 控制，當 app 大起來後，會漸漸變得複雜，redux 簡單來說就是讓 root component 集中管理整個 app 的 state，讓 state 變得容易控制、取得。

redux-persist

大部分時候，需要將 app 的狀態保存起來，總不能每次打開 app 都是教學歡迎畫面吧？這個套件將 app 的狀態儲存進手機內，打開 app 時讀取資料。

react-navigation

app 是由多個畫面組成，這個套件管理所有的畫面設定，包括呈現、動畫等等，另外還提供 SafeView，讓 app 預留狀態欄空間等等。

UI 套件

  • react-native-elements
  • native-base

非常適合簡單元件的加入，但在兩種平台可能看起來不一樣，需要多加注意。

RN StyleSheet 和 Web CSS 之區別

Variable Style

※ 使用 StyleSheet 的原因是它會產生一個 style ID 傳入元件，防止每次 render 時都產生一個新的 style 物件。

元件排版

  • 螢幕畫面長寬需引入 Dimensions library。

  • 基本 block 元件如 View 的內容排版都是 flexbox，因此不須加入 display: flex，且常常要設定 flex，也就是 CSS 的 flex-grow。

  • Position 則沒有 fixed，使用 absolute 就相當夠用。

  • 根據平台設定不同的 style

小提醒

Overflow

  • Android 中元件預設 overflow 為 hidden 且無法更改。
  • iOS 則是預設 visible

    Status Bar

    畫面中需要為狀態列預留空間，或是客製化狀態列，react-native 中有 SafeAreaView 可以使用。
  • Android 預設為黑色
  • iOS 預設為透明

    平台特色

  • Android 之元件多為 Material design，且有實體返回鍵需要處理。
  • iOS 則有許多漸進動畫，通常是以畫面右滑做為返回手勢

但畫面之設計並沒有硬性規定，可以自由發揮。
